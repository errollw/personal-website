Computer graphics & image processing

Work for this course includes:

  1. Written exercises – from the course website and textbook.
  2. Practical exercises – learn the algorithms better by coding them.
  3. Past exam questions – discover what is expected in the exam.

Textbook

I recommend acquiring a copy of Fundamentals of Computer Graphics [FCG] – Shirley, P. & Marschner, S. (2009). This is the main textbook for the course and it covers the course's concepts and algorithms in detail, providing pseudo code and examples.

Some of the written supervision exercises are taken from the book.

Check for a copy in your college or the lab library, or try to find one online.

Supervision 1

Written exercises – Introduction and simple rendering

  1. Why is it better to look at faint stars and comets slightly off-centre rather than looking directly at them?
  2. Why do we use square pixels? Would hexagonal pixels be better? What about triangles? Do you see any difficulties building graphics hardware with these other two schemes?
  3. What are the ray parameters of the intersection points between ray (1,1,1) + t(−1,−1,−1) and the sphere centered at the origin with radius 1? [FCG p89]
  4. Why do most highlights on plastic objects look white, while those on gold metal look gold? [FCG p242]

Practical work – write a ray tracer

Write a simple ray-tracer. Start with raytracing a sphere and shading it with Phong illumination. Then experiment with:

  • Multiple lights of different colours,
  • More objects like triangles or blocks,
  • Reflection and shadows.

All you need is in the .zip file. The textbook section on Ray Tracing should be very helpful [FCG p69].

Please submit images of your results as .png or .jpg.

Supervision 2

Written exercises – Graphics pipeline, algorithms for 2D and 3D

  1. Why do we use matricies in graphics, why are they useful?
  2. Bézier cubics – Derive the conditions necessary for two Bézier curves to join with
    1. just C0-continuity
    2. C1-continuity
    3. C2-continuity
    What is difficult about getting three Bézier curves to join in sequence with C2-continuity at the two joins?
  3. Break down the following 2D lines into a BSP-tree, splitting them if necessary:
    • (0,0)–(2,2);   (3,4)–(1,3);   (1,0)–(-3,1);   (0,3)–(3,3);   (2,0)–(2,1)
  4. Draw and describe a simple diagram of the rendering pipeline.
  5. What is a vertex shader? What is a fragment shader? Describe the difference between uniform, attribute and varying variables, and give examples of each.
  6. Compare and contrast texture mapping, bump mapping, and displacement mapping.

Practical work – explore OpenGL

Render a scene using OpenGL, including multiple objects, shading, and texture mapping. Move onto adding:

  • Bump or displacement mapped objects,
  • Stylised rendering (e.g. toon shading),
  • Animations, perhaps with the vertex shader.

All you need to get started is in the .zip file. The online GL Tutor could be helpful for prototyping GLSL code. Please submit images of your results as .png or .jpg.

Notes on practical exercises

I strongly encourage you to do as many practical exercises as possible, as algorithms are frequently examined and there is no better way to learn them than to code them yourself! If you struggle with the code, please let me know ahead of the supervision.

Practical exercise 1

For the first supervision you will write a simple ray tracer. You can use whatever programming language or tools for this as you wish, but I suggest that you use Javascript. You should have an HTML5-compatible browser which can set pixels in a Canvas element.

I have packaged all you need in a .zip file that contains:

  • raytracer.html – open this in a web browser to run the Javascript and render the image.
  • raytracer.js – modify this template to write your ray tracer.
  • vector.js – this is a vector maths library, no need to modify it.

The file you should modify is raytracer.js. Please rename this to crsid_raytracer.js with your CRSid and email it to me. Beware, if you rename this file without changing it in raytracer.html, it will stop working.

Practical exercise 2

For the second supervision you will write create a scene using OpenGL. As before, you can use whatever tools you want, but I suggest that you use WebGL and three.js. Modern browsers are WegGL-compatible, and some libraries exist which makes things easier for you.

three.js abstracts away some of the headaches of WebGL, while still giving you an idea of how the pipeline works. Some tutorials are available here.

I have packaged all you need in a .zip file that contains:

  • wegbl.html – this file contains the GLSL code for the vertex shader, fragment shader, and Javascript code for setting up the scene. Modify the code in this file.
  • three.min.js – this library abstracts away lots of the headaches of WebGL, while still giving you an idea of how the pipeline works.
  • OrbitControls.js – this library provides mouse controls for rotating the camera.

The file you should modify is wegbl.html.
